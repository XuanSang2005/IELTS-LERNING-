# Coding Conventions

**Analysis Date:** 2026-04-22

Meridian is an editorial-brand IELTS platform. Every convention below is enforced either by tooling (ESLint 9 flat, Prettier, TypeScript strict, Husky) or by `CLAUDE.md` (brand/design rules). Treat both as hard gates.

## Naming Patterns

**Directories:**
- Feature folders: **kebab-case**, singular domain noun — `frontend/src/features/practice/`, `frontend/src/features/landing/`, `frontend/src/features/auth/`, `frontend/src/features/tests/`, `frontend/src/features/vocabulary/`
- Sub-folders inside a feature are fixed: `components/`, `hooks/`, `utils/`, `data/`
- Backend NestJS modules: kebab-case domain noun — `backend/src/auth/`, `backend/src/practice/`, `backend/src/lessons/`, `backend/src/test-submissions/`

**Files:**
- React components: **PascalCase.tsx** — `Header.tsx`, `PhaseCard.tsx`, `LoginForm.tsx`, `BackgroundOrnaments.tsx`, `SpecimenCard.tsx`
- Hooks / utils / stores / schemas: **kebab-case.ts** — `practice-queries.ts`, `practice-mutations.ts`, `auth-store.ts`, `test-runner-store.ts`, `api-client.ts`, `query-client.ts`, `session-planner.ts`
- TanStack Router route files: `routes/<segment>.tsx`, dot-separated for nesting — `routes/app.session.index.tsx`, `routes/tests.$testId.results.tsx`, `routes/__root.tsx`
- NestJS files: `<domain>.<role>.ts` — `auth.controller.ts`, `auth.service.ts`, `auth.module.ts`, `jwt.strategy.ts`, `zod-validation.pipe.ts`, `jwt-auth.guard.ts`, `current-user.decorator.ts`
- Generated files: `routeTree.gen.ts` (TanStack Router) — **never hand-edit**, regenerated by `npm run tsr:generate`

**Code identifiers:**
- Functions / variables: **camelCase** — `apiFetch`, `useAuthStore`, `todayIso`, `patchTodayLog`
- React components / classes / TS interfaces / types / Zod schemas: **PascalCase** — `LoginForm`, `ApiError`, `AuthenticatedUser`, `NoticingItemSchema`
- Zod schemas carry the `Schema` suffix, the inferred type drops it: `NoticingItemSchema` → `type NoticingItem = z.infer<typeof NoticingItemSchema>` (see `shared/schemas/practice.ts:34-53`)
- DTO schemas: `<Verb><Noun>DtoSchema` — `AddNoticingItemDtoSchema`, `UpdateNoticingItemDtoSchema`, `ReviewItemDtoSchema` (see `shared/schemas/practice.ts:55-82`)
- React Query keys: `const <feature>StateKey = [...] as const` — `practiceStateKey = ['practice', 'state'] as const` (see `frontend/src/features/practice/hooks/practice-queries.ts:13`)
- Zustand stores: exported hook `use<Name>Store` — `useAuthStore` (see `frontend/src/stores/auth-store.ts:12`)
- Persisted-storage keys: versioned — `meridian-<feature>-v1` (see `frontend/src/stores/auth-store.ts:21`)
- CSS tokens in `@theme`: `--color-<name>`, `--font-<name>` (kebab-case, see `frontend/src/styles/globals.css:3-27`)

## Code Style

**Formatting (Prettier 3.8.3):**
- Root config: `.prettierrc.json` (covers `shared/` and repo-root markdown/json)
- Frontend config: `frontend/.prettierrc` (adds `prettier-plugin-tailwindcss` for class ordering)
- Backend config: `backend/.prettierrc`
- All three share identical base settings:
  - `"semi": false` — **no semicolons**
  - `"singleQuote": true`
  - `"trailingComma": "all"`
  - `"printWidth": 100`
  - `"tabWidth": 2`, `"useTabs": false`
  - `"arrowParens": "always"`
  - `"bracketSpacing": true`
  - `"endOfLine": "lf"` (backend adds `{ endOfLine: 'auto' }` override in its ESLint Prettier rule — `backend/eslint.config.mjs:32`)
- Ignores: `.prettierignore` and `frontend/.prettierignore` — never format `**/routeTree.gen.ts`, `dist`, `build`, `node_modules`, lock files, `.env*`, `.husky/_`, `coverage`, `**/.tanstack`

**Linting (ESLint 9 flat config):**
- Three configs, each tuned to its runtime — **never merge them**:
  - `eslint.config.mjs` — repo root, scoped to `shared/**/*.ts` only; type-aware via `./shared/tsconfig.json`; globally ignores `frontend`, `backend`, `dist`, `build`, `.husky`, `**/routeTree.gen.ts`
  - `frontend/eslint.config.js` — `@eslint/js` + `typescript-eslint` + `eslint-plugin-react-hooks` (flat recommended) + `eslint-plugin-react-refresh` (Vite preset). Ignores `dist` and `src/routeTree.gen.ts`. Route files under `src/routes/**/*.{ts,tsx}` have `react-refresh/only-export-components` disabled because TanStack Router exports a `Route` config alongside the component.
  - `backend/eslint.config.mjs` — `@eslint/js` + `typescript-eslint` **recommendedTypeChecked** + `eslint-plugin-prettier/recommended`. `@typescript-eslint/no-explicit-any` is off; `no-floating-promises` and `no-unsafe-argument` are warnings.
- Run via workspace scripts: `npm run lint` (root) fans out to `npm --prefix frontend run lint` and `npm --prefix backend run lint`. `npm run lint:fix` does the same with `--fix`.

**TypeScript:**
- Frontend (`frontend/tsconfig.app.json`): `strict: true`, `noUnusedLocals`, `noUnusedParameters`, **`erasableSyntaxOnly: true`**, `noFallthroughCasesInSwitch`, `verbatimModuleSyntax`, `moduleResolution: "bundler"`, `target: "es2023"`, `jsx: "react-jsx"`, `noEmit: true`
- `erasableSyntaxOnly` means: **no parameter-property constructors, no enums, no namespaces** — stick to `class X { constructor(private foo: string) {} }` → use plain field assignments; use `as const` unions or Zod enums instead of TS enums
- Backend (`backend/tsconfig.json`): `strictNullChecks`, `noImplicitAny`, `strictBindCallApply`, `noFallthroughCasesInSwitch`, `emitDecoratorMetadata`, `experimentalDecorators`, `module: "nodenext"`, `target: "ES2023"` (Nest needs decorator metadata, so no `erasableSyntaxOnly` there)
- Shared (`shared/tsconfig.json`): `strict`, `target: ES2022`, `moduleResolution: Bundler`
- Typecheck: `npm run typecheck` at root → `npm --prefix frontend run typecheck` (runs `tsr generate` first via `pretypecheck`, then `tsc -b --noEmit`) + `npm --prefix backend run build` (nest build produces emitted output as the check)

## Import Organization

**Order (observed across the codebase — not enforced by a plugin, but consistent):**
1. Node built-ins (`node:path`, `node:fs`)
2. Third-party packages (`react`, `@tanstack/*`, `zod`, `framer-motion`, `zustand`)
3. `@shared/*` shared schemas
4. `@/*` internal frontend code
5. Relative imports (`./...`, `../...`)

Example (`frontend/src/features/practice/hooks/practice-mutations.ts:1-16`):
```ts
import { useMutation, useQueryClient } from '@tanstack/react-query'
import type { AddNoticingItemDto, BandLevel, ... } from '@shared/schemas/practice'
import { apiFetch } from '@/lib/api-client'
import { applySm2 } from '@/features/practice/utils/sm2'
import { isoDaysFromNow, todayIso } from '@/features/practice/utils/dates'
import { practiceStateKey } from './practice-queries'
```

**Path aliases:**
- Frontend: `@/*` → `frontend/src/*`, `@shared/*` → `shared/*` (declared in `frontend/tsconfig.app.json:27-29` and mirrored in `frontend/vite.config.ts:19-24`)
- Backend: `@shared/*` → `../shared/*` (declared in `backend/tsconfig.json:17-19`)
- Never use deep relative paths like `../../../shared/...` — always use `@shared/*`
- Never use deep relative paths inside the frontend — always use `@/*`

**Type-only imports:**
- Use `import type { ... }` whenever importing only types (required because `verbatimModuleSyntax: true`)
- Example: `import type { AuthResponse, User } from '@shared/schemas/auth'` (see `frontend/src/stores/auth-store.ts:3`)

## Zod as Source of Truth

Schemas come first; types are always inferred. **Never hand-author a `type` or `interface` for data that crosses the network or localStorage boundary.**

```ts
// shared/schemas/practice.ts:34-53
export const NoticingItemSchema = z.object({
  id: z.string(),
  text: z.string().min(1),
  category: DisciplineSchema,
  // ...
})
export type NoticingItem = z.infer<typeof NoticingItemSchema>
```

**Where schemas live:**
- Shared between frontend and backend → `shared/schemas/*.ts` (e.g. `auth.ts`, `practice.ts`, `lesson.ts`, `study.ts`, `submission.ts`, `test.ts`, `vocabulary.ts`)
- Frontend-only (rare) → `frontend/src/schemas/*.ts` (currently only `practice.ts` as a re-export stub)
- Backend Mongoose schemas: separate folder `backend/src/<module>/schemas/` — **distinct from Zod DTOs**; Zod validates input, Mongoose defines persistence shape

**Persistence rehydration pattern** (Zustand `persist` middleware, `frontend/src/stores/auth-store.ts:20-35`):
- Version the storage key: `name: 'meridian-auth-v1'`
- Validate on `merge` with the schema: `AuthResponseSchema.safeParse(...)`
- On invalid → fall back to `current` (initial) state, never throw

**Backend DTO validation** (`backend/src/common/zod-validation.pipe.ts`):
```ts
@Post('register')
@UsePipes(new ZodValidationPipe(RegisterDtoSchema))
async register(@Body() dto: RegisterDto): Promise<AuthResponse> {
  const res = await this.auth.register(dto)
  return AuthResponseSchema.parse(res) // response is ALSO validated before returning
}
```
Controllers **always** parse responses with the Zod schema before returning (see `backend/src/auth/auth.controller.ts:26-36`). This keeps the wire contract honest in both directions.

## HTTP Pattern

**Use `apiFetch<T>` from `frontend/src/lib/api-client.ts`. Never add `axios`, `ky`, or another HTTP client.**

```ts
// frontend/src/lib/api-client.ts:27-56
export async function apiFetch<T>(path: string, init: RequestInit = {}): Promise<T>
```

Behaviour:
- Prepends `env.VITE_API_URL` unless the path already starts with `http`
- Auto-sets `Content-Type: application/json` when body is a string
- Auto-sets `Accept: application/json`
- Auto-attaches `Authorization: Bearer <token>` from `useAuthStore`
- On 401 → clears auth, redirects to `/login?redirect=<encoded>`, throws `ApiError`
- On non-2xx → throws `ApiError(status, statusText, body)`
- Parses response JSON automatically based on Content-Type

**Queries** wrap `apiFetch` inside TanStack Query (`frontend/src/features/practice/hooks/practice-queries.ts`):
```ts
export const practiceStateKey = ['practice', 'state'] as const
export function usePracticeState() {
  const token = useAuthStore((s) => s.token)
  return useQuery({
    queryKey: practiceStateKey,
    queryFn: () => apiFetch<PracticeStateShape>('/practice/state'),
    enabled: Boolean(token),
    staleTime: 60_000,
  })
}
```

**Mutations** use optimistic updates with rollback + invalidate-on-settle (see `frontend/src/features/practice/hooks/practice-mutations.ts:42-89`):
```ts
onMutate: async (dto) => {
  await qc.cancelQueries({ queryKey: practiceStateKey })
  const previous = qc.getQueryData<State>(practiceStateKey)
  // ...apply optimistic patch with qc.setQueryData...
  return { previous }
},
onError: (_err, _dto, ctx) => {
  if (ctx?.previous) qc.setQueryData(practiceStateKey, ctx.previous)
},
onSettled: () => {
  void qc.invalidateQueries({ queryKey: practiceStateKey })
},
```

## Forms Pattern

**Use TanStack Form + Zod. Never add `react-hook-form`, `formik`, or `final-form`.**

Example (`frontend/src/features/auth/components/LoginForm.tsx:19-33`):
```ts
const form = useForm({
  defaultValues: { email: '', password: '' },
  validators: { onChange: LoginDtoSchema }, // Zod schema as the validator
  onSubmit: async ({ value }) => { /* ... */ },
})
```

Rules:
- Validators come from the same Zod schema the backend DTO uses (`@shared/schemas/*`)
- Fields are rendered via `<form.Field name="..." children={(field) => ...}>`
- Submit state read via `<form.Subscribe selector={...} children={...}>`
- Always `e.preventDefault(); e.stopPropagation(); void form.handleSubmit()` in the `onSubmit`
- Wrap controlled inputs in a feature-local component (e.g. `AuthInput`, `AuthSubmit`, `FormErrorNote` in `frontend/src/features/auth/components/`) so the editorial styling stays consistent

## State Pattern

**Server state → TanStack Query. Client state → Zustand. One store per concern.**

Current stores (`frontend/src/stores/`):
- `auth-store.ts` — `useAuthStore` (token + user, persisted to `localStorage` as `meridian-auth-v1`)
- `test-runner-store.ts` — `useTestRunnerStore` (in-flight test session state)

Rules:
- One file per store, kebab-case, suffixed `-store.ts`
- Export the hook `use<Name>Store`; never export the raw store object
- Select narrowly — `useAuthStore((s) => s.token)` — not `useAuthStore()`
- Use `persist` middleware with versioned key and Zod-validated `merge` (see pattern in `frontend/src/stores/auth-store.ts`)
- Server-owned data (profile, noticing items, daily logs, submissions, tests, lessons) lives in TanStack Query cache, **not** in Zustand — the cache is the source of truth, mutations optimistically update it

## Anti-patterns (hard bans — enforced by review, not tooling)

From `CLAUDE.md` §Things to avoid and §Anti-patterns:

- **No `react-router-dom`** — routing is TanStack Router with file-based routes under `frontend/src/routes/`
- **No `axios`, `ky`, `got`** — use `apiFetch<T>()` from `frontend/src/lib/api-client.ts`
- **No `react-hook-form`, `formik`, `final-form`** — forms are TanStack Form + Zod
- **No `react-i18next` or any i18n library** — UI is English-only by policy; Vietnamese appears only as per-phrase popover translations driven by a Claude Haiku API call
- **No pure `#000`** — always use `ink` (`#141210`) or `ink-warm` (`#2A2622`)
- **No `rounded-2xl` or softer** — corners are square or ≤2px radius
- **No purple/blue gradients, glassmorphism, neon, 3D blobs**
- **No Inter / Roboto / Arial / system fonts** — only Fraunces, Geist, JetBrains Mono
- **No generic black drop shadows** — shadows are claret-tinted, e.g. `shadow-[0_10px_25px_-8px_rgba(107,31,26,0.35)]`
- **No hardcoded pricing** — all prices flow from env vars (`PRO_MONTHLY_PRICE_VND`, `PRO_COHORT_PRICE_VND`) and a `GET /pricing` endpoint; use the placeholder `{PRICING_TBD}` in copy until locked
- **No gamification** — no streaks, XP, levels, badges, confetti, fire emoji, "Don't break your streak!" modals, or public leaderboards
- **No emoji in UI copy** (arrows like `→` `↗` are fine; these are typographic, not emoji)
- **No centre-aligned long body text**
- **No SaaS copy** ("Get started free", "Sign up now", "Click here", "Learn more", "Read more" are banned)
- **No Vietnamese in UI strings** (except the on-demand translation popover content)
- **No human teacher grading** — feedback is AI only
- **No parameter-property constructors** (`erasableSyntaxOnly`) — spell out field assignments
- **No `postcss.config.js` or `autoprefixer`** — `@tailwindcss/vite` handles both
- **No `--no-verify`** — Husky hooks are non-negotiable; don't bypass
- **No committing `.env` or `.env.local`** — both are in `.gitignore` and `.prettierignore`
- **No feature-specific components in `frontend/src/components/`** — only truly shared primitives live there (`components/layout/Header.tsx`, `components/ornaments/*`, `components/ui/Polaroid.tsx`). Everything else belongs in `features/<name>/components/`
- **No types co-located with features** — types are always inferred from Zod schemas in `shared/schemas/` or (rarely) `frontend/src/schemas/`
- **Don't amend `routeTree.gen.ts` by hand** — run `npm run tsr:generate`

## Error Handling

**Frontend:**
- `ApiError` class from `frontend/src/lib/api-client.ts:4-16` carries `status`, `statusText`, and `body: unknown`
- 401 is handled centrally: auth cleared, full `window.location.href` redirect to `/login?redirect=<current>` so route guards re-evaluate cleanly (`api-client.ts:18-25`)
- Per-feature copy mapping: `authErrorCopy(err, 'login' | 'register')` in `frontend/src/lib/auth.ts` turns raw errors into editorial-voice messages
- Form-level errors are rendered through a small dedicated component (`FormErrorNote`), never `alert()` or `toast`

**Backend:**
- Zod validation failures → `BadRequestException({ message: 'Validation failed', issues })` from `ZodValidationPipe` (see `backend/src/common/zod-validation.pipe.ts:9-13`)
- Domain errors use Nest exception classes: `NotFoundException`, `UnauthorizedException`, `ForbiddenException` (see `backend/src/auth/auth.controller.ts:42`)
- Never swallow errors silently; never `console.log` a secret-bearing payload

## Logging

- No logging framework (no Winston, Pino, Bunyan). Plain `console.log` / `console.error`.
- Backend boot logs a single line: `console.log('[meridian-api] listening on :${port}')` (`backend/src/main.ts:30`)
- Frontend env-validation failures log via `console.error('Invalid environment variables:', ...)` and throw before the app boots (`frontend/src/lib/env.ts:9-12`)
- **Never** log the raw `Authorization` header, `token`, password, or Anthropic API response bodies
- Frontend devtools only mount in dev: `import.meta.env.DEV && <TanStackRouterDevtools />` (see `frontend/src/routes/__root.tsx:41-46`)

## Comments

**When to comment:**
- Explain *why* a non-obvious decision was made, especially around brand/editorial constraints or framework quirks
- Good example (`frontend/eslint.config.js:23-26`): explains why `react-refresh/only-export-components` is off for route files (TanStack Router exports `Route` config alongside the component)
- Good example (`frontend/src/lib/api-client.ts:21`): explains why `window.location.href` is used instead of router navigation on 401 ("route guards re-evaluate cleanly")
- Pedagogy/business comments tie code to `CLAUDE.md` concepts (`profile.phase`, `currentWeek`, `meridian-auth-v1`)

**JSDoc:**
- Used sparingly on Zod schema fields to document meaning — `/** Where the item came from: 'system' = seeded/imported, 'user' = captured by the learner. */` (see `shared/schemas/practice.ts:40`)
- Not required on every function; prefer self-documenting names and small functions

## Function Design

- Small, single-purpose. Most utils in `frontend/src/features/practice/utils/` are 10-40 lines.
- Destructured named-object parameters for mutations with 2+ inputs: `mutationFn: ({ id, quality }: { id: string; quality: ReviewQuality }) => ...`
- Pure helpers live in `utils/` folders — `sm2.ts`, `dates.ts`, `session-planner.ts`. No React, no side effects.
- Prefer arrow functions for callbacks/handlers; use `function` declarations for top-level hooks and components (both styles coexist, neither is enforced)

## Module Design

**Exports:**
- **Named exports only** for components, hooks, schemas, stores. Default exports are only used where a framework requires it (none currently — even routes use `createFileRoute` / `createRootRoute` with a named `Route` const).
- One default-behaviour concept per file: one component, one hook group, one store.

**Barrel files:**
- **None.** There is no `index.ts` re-exporting a folder. Import the specific file — `@/features/practice/hooks/practice-queries` — not the folder.
- Rationale: keeps TanStack Router's auto-code-splitting honest and avoids accidental circular imports.

## Commit Hook Pipeline

Husky v9 is installed via root `package.json` `"prepare": "husky"` script.

**`.husky/pre-commit`** (one line):
```sh
npx lint-staged
```

**`.husky/pre-push`** (one line):
```sh
npm --prefix frontend run typecheck
```

**`.lintstagedrc.cjs`** — each workspace runs its own `eslint` + `prettier` binaries so per-project plugins (e.g. `prettier-plugin-tailwindcss` on the frontend) resolve correctly:

| Glob | Commands |
| --- | --- |
| `frontend/src/**/*.{ts,tsx}` | `cd frontend && ./node_modules/.bin/eslint --fix <files>` then `... prettier --write <files>` |
| `frontend/src/**/*.{css,json,md}` | `cd frontend && ./node_modules/.bin/prettier --write <files>` |
| `backend/src/**/*.ts` | `cd backend && ./node_modules/.bin/eslint --fix <files>` then `... prettier --write <files>` |
| `backend/src/**/*.{json,md}` | `cd backend && ./node_modules/.bin/prettier --write <files>` |
| `shared/**/*.{ts,json,md}` | `./frontend/node_modules/.bin/prettier --write <files>` (from repo root) |

**Root scripts** (`package.json`):
- `npm run format` — prettier-writes `shared/**`, then forwards to `frontend` and `backend`
- `npm run format:check` — CI-safe check of `shared/` and repo-root `*.{json,md}`
- `npm run lint` / `lint:fix` — fans out to both workspaces
- `npm run typecheck` — frontend `tsc -b --noEmit` + backend `nest build`

**Never bypass:** `--no-verify` on commit/push is forbidden (`CLAUDE.md` §Things to avoid). If a hook fails, fix the underlying issue and make a new commit.
